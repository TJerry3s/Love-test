# 恋爱倾向测试网页产品PRD方案

## 一、产品概述

### 1.1 产品名称
恋爱倾向测试

### 1.2 产品定位
一款支持手机端与PC端网页适配的趣味恋爱风格测试工具，面向对恋爱心理学感兴趣、希望了解自身恋爱倾向的年轻群体，通过标准化测试流程、趣味性的结果分析，帮助用户发现自己在恋爱中的行为模式、情感需求和相处偏好。

### 1.3 核心目标
- 为用户提供便捷、有趣的恋爱倾向在线测试，保障测试流程的流畅性和结果的趣味性；
- 帮助用户清晰认知自己的恋爱风格，识别在亲密关系中的行为模式；
- 传递恋爱心理学知识，引导用户建立健康的恋爱观；
- 实现产品在手机端与PC端的流畅适配，覆盖不同场景下的用户使用需求。

### 1.4 目标用户
- 核心用户：18-35岁年轻群体，对恋爱关系感兴趣，希望了解自身恋爱风格的单身或恋爱中人群；
- 潜在用户：对心理学感兴趣，希望了解恋爱心理学知识的群体；情侣双方希望通过测试增进了解。

## 二、用户需求分析

### 2.1 核心需求
- 能够随时随地通过手机或电脑完成恋爱倾向测试，测试流程简单有趣，耗时合理；
- 测试结果有趣、详细，包含清晰的恋爱类型解读，避免专业术语过于晦涩；
- 根据测试结果获得针对性的恋爱建议，知道如何改善自己的恋爱关系或寻找合适的伴侣；
- 保障个人测试数据的安全性与隐私性，避免信息泄露。

### 2.2 次要需求
- 可以保存测试结果，方便分享给朋友或恋人；
- 获取有趣的恋爱心理学知识，辅助提升恋爱技巧；
- 测试过程中可随时修改答案。

## 三、产品核心功能规划

### 3.1 用户端核心功能

#### 3.1.1 测试前准备模块
- 无需登录，点击链接进入测试首页；
- 展示测试说明、测试维度、隐私保护说明。

#### 3.1.2 测试过程模块
- 题目展示：按顺序呈现20道标准题目，支持手机端与PC端适配的排版；
- 作答方式：每道题提供5个选项（1=完全不符合，2=比较不符合，3=一般，4=比较符合，5=完全符合），用户点击选项即可完成作答；
- 进度提示：页面显示当前测试进度（如"第5题/共20题"）；
- 答案修改：支持返回上一题修改答案；
- 自动跳转：选择答案后自动跳转到下一题（可手动控制）。

#### 3.1.3 测试结果模块
- 结果生成：用户完成所有题目后，系统自动计算各维度得分，生成趣味性测试报告；

- 结果解读：
  - 核心结论：明确告知用户的恋爱类型（如浪漫激情型、理性务实型等）；
  - 维度详情：分维度解读得分含义，说明各维度对应的恋爱倾向；
  - 五维雷达图：可视化展示5个维度的得分分布；

- 建议输出：
  - 择偶建议：根据用户的恋爱类型，推荐适合的伴侣类型；
  - 相处建议：提供改善恋爱关系的具体建议；
  - 自我成长：提供个人成长的建议；

- 最佳匹配类型：根据用户的恋爱类型，推荐最适合匹配的恋爱类型；

- 结果保存与分享：支持生成报告链接或截图分享。

## 四、产品交互与设计要求

### 4.1 适配要求
- 手机端：支持主流移动端浏览器（微信内置浏览器、Chrome、Safari等），适配不同屏幕尺寸（3.5-6.7英寸），页面布局简洁，操作按钮大小适宜（便于手指点击），避免过多滚动操作；
- PC端：支持主流桌面浏览器（Chrome、Firefox、Edge、Safari等），页面分辨率适配（最低1366×768），布局合理，充分利用屏幕空间，提升信息展示效率。

### 4.2 交互原则
- 流程简洁：测试流程清晰，减少不必要的操作步骤，用户从进入页面到开始测试的步骤不超过2步；
- 反馈及时：用户操作后（如作答、提交）及时给出反馈提示（如按钮状态变化）；
- 引导明确：关键节点（如测试说明、结果解读）提供清晰的引导文案，避免用户困惑；
- 容错性强：对用户可能的误操作进行防护，提供撤销、确认等功能。

### 4.3 技术选型
- 使用前端技术解决，保证代码可以用WebStorm 2025.2.1运行、修改；
- 如果涉及到数据存储的技术也用前端技术完成，不使用任何后端技术；
- 使用Vue 3框架，html2canvas用于截图保存。

### 4.4 设计风格
整体风格简洁、温馨、有趣，色调以柔和的粉色、珊瑚色为主（传递温暖、浪漫的感受），避免使用过于鲜艳或压抑的色彩；字体清晰易读，专业术语搭配通俗解释，提升用户理解度。

## 五、测试内容设计

### 5.1 测试维度
基于爱情三元论（激情、亲密、承诺）及恋爱心理学研究，设计5个测试维度：

| 维度 | 说明 |
|------|------|
| 激情型 | 反映在恋爱中对心动体验、浪漫感受和外在吸引的重视程度 |
| 理智型 | 反映在恋爱中对现实条件、未来规划和理性决策的重视程度 |
| 奉献型 | 反映在恋爱中愿意为对方付出的程度和自我牺牲的倾向 |
| 依赖型 | 反映在恋爱中对情感陪伴和依赖的需求程度 |
| 独立型 | 反映在恋爱中保持自我空间和独立性的需求程度 |

### 5.2 题目设计
- 测试包含20道题目，每个维度4道题目；
- 题目表述简洁明了，避免歧义；
- 题目覆盖恋爱中的常见场景和感受。

### 5.3 答案选项
采用5点李克特量表：
1. 完全不符合
2. 比较不符合
3. 一般
4. 比较符合
5. 完全符合

### 5.4 结果类型
根据5个维度的得分组合，定义9种恋爱类型：

| 类型名称 | 典型特征 | 维度组合特征 |
|---------|---------|-------------|
| 浪漫激情型 | 追求心动和浪漫体验 | 激情型高、理智型低 |
| 理性务实型 | 注重现实条件和规划 | 理智型高、激情型低 |
| 无私奉献型 | 愿意无条件付出 | 奉献型高、独立型低 |
| 甜蜜依恋型 | 渴望亲密陪伴 | 依赖型高、独立型低 |
| 独立平等型 | 保持自我空间 | 独立型高、依赖型低 |
| 完美平衡型 | 各维度均衡发展 | 5个维度得分接近 |
| 理智浪漫型 | 理性与激情并存 | 激情型高、理智型高 |
| 温和陪伴型 | 温和包容的相处方式 | 奉献型高、独立型高 |
| 谨慎保守型 | 各维度得分偏低 | 5个维度得分均偏低 |

## 六、结果页内容设计

### 6.1 结果页结构（自上而下）
1. 顶部标题区：产品名称、测试时间
2. 主要恋爱类型：类型名称、emoji、标签、一句话描述
3. 五维雷达图：可视化展示5个维度得分
4. 维度得分详情表：各维度的得分、等级、类型标签
5. 类型详细解读：类型特征、优点、注意事项
6. 恋爱建议：择偶建议、相处建议、自我成长建议
7. 最佳匹配类型：推荐的匹配类型和匹配度说明
8. 保存/分享区：保存为图片按钮、重新测试按钮

## 七、数据安全与隐私保护

### 7.1 数据安全
- 每个用户的测试数据仅保存在本地浏览器中；
- 不收集或上传用户的个人信息；
- 不使用任何后端技术存储数据。

### 7.2 隐私保护
- 测试结果仅供用户自我探索和娱乐；
- 用户可以选择是否分享测试结果；
- 明确告知用户数据仅存储在本地。

## 八、非功能需求

### 8.1 性能要求
- 页面加载速度：手机端页面首次加载时间≤3秒，PC端≤2秒；测试过程中页面切换流畅，无明显卡顿；
- 结果生成速度：用户完成测试后，结果报告生成时间≤2秒。

### 8.2 稳定性要求
- 系统运行稳定，出现异常情况时给出友好提示；
- 确保用户数据不丢失。

### 8.3 兼容性要求
- 浏览器兼容性：支持主流浏览器最新版本及上一个版本（手机端：微信浏览器、Chrome、Safari；PC端：Chrome、Firefox、Edge、Safari）；
- 设备兼容性：手机端适配主流品牌（华为、苹果、小米、OPPO、vivo等）不同型号手机；PC端适配台式机、笔记本电脑不同屏幕尺寸。

## 九、产品演进规划

### 9.1 第一阶段（当前版本）
- 基础测试功能：20道题目，5个维度，9种恋爱类型；
- 结果展示：雷达图、维度详情、类型解读、恋爱建议、匹配类型；
- 保存功能：支持保存为图片。

### 9.2 第二阶段（未来规划）
- 增加情侣对比功能：两个用户的测试结果对比分析；
- 增加历史记录功能：保存多次测试记录，对比变化；
- 增加分享功能：生成专属分享链接。
